Planning Workflow

SCRIPTA Documentation

Planning is the core of the system. SCRIPTA treats a book as a progressively refined structure that moves from macro intent to chapter progression and finally to symbolic micro blocks.

All plan files use a shared CNL line format: @identifier verb on its own line, followed by fixed labeled lines and optional natural-language hints on separate lines until the next @identifier header. Scripts parse the headers and labels; LLMs read the full block as text. 
flowchart TD
        brief[User brief] --> symbolic[Symbolic plan generation]
        symbolic --> macro[MacroPlan]
        macro --> chapter[ChapPlan]
        chapter --> micro[MicroPlan]
        micro --> cnl[CNLEnh]
        cnl --> draft[ChapGen]

Symbolic seed generation

Every planning skill must begin from symbolic seed generation. Skills are expected to call CLI-style tooling that writes the first iteration of macro, chapter, or micro plan files from explicit parameters such as work size, genre profile, structural constraints, and random seed. 

scripta-plan seed --stage macro \
  --book-id judge-memory \
  --work-form novel \
  --baseline-profile detective-police \
  --chapter-count 18 \
  --scene-density medium \
  --narrative-model three-act \
  --seed 1042 \
  --placeholder-style symbolic-id

The seed files may contain typed placeholders such as {{character:PERSON_001}} or {{location:LOCATION_001}}, and they may use references such as $character-protagonist-001 or $location-primary when a block points back to something already defined. The macro stage also emits a symbolic entity map so later stages know which generic id corresponds to protagonist, counterpart, pressure figure, or primary location. Seed files must already contain valid command identifiers, enumerated classifier values, and structural constraints. Later stages refine these files instead of replacing them.

MacroPlan

MacroPlan defines the book concept before chapter writing begins. It must produce the central idea, theme, wisdom, narrative structure, narrative model, blueprint, plot elements, characters, and worldbuilding. 

@central-idea define
hook-pattern: costed-power
tension-source: mixed
naming-state: placeholder
hook: a judge can rewrite verdicts but loses a personal memory every time
protagonist: $character-protagonist-001
desire: save innocent defendants
opposition: the system and the cost of self-erasure
stakes: justice, identity, trust
story-question: how much truth is worth if every act of justice erases the self?
The idea should feel like psychological drama first and speculative mechanism second.

@theme define
topic: identity-self
moral-shape: dilemma
thematic-question: is truth still right when it destroys the person who carries it?
thematic-statement: concealment protects comfort for a time but corrodes both justice and identity

WisdomWhat reflective depth should the book generate?Cognitive, emotional, moral, reflexive, experiential aims.

ChapPlan

ChapPlan maps the macro skeleton into a chapter sequence. Each chapter must have a narrative function, a local conflict, an input state, an output state, and a clear contribution to the overall transformation of the book.

MicroPlan

MicroPlan converts chapter intent into symbolic block arrangements. It must keep the output abstract enough for deterministic processing while still encoding ordering, tension progression, and local function.

CNLEnh

CNLEnh does not write final chapters. It enriches symbolic plans with controlled textual hints that tell the drafting stage what effect, tone, and local emphasis to aim for. It is also the normal stage where typed placeholders become concrete names, places, and institutions while $identifier references remain available as planning links.

The refinement stage must now keep scene packets distinct enough for long-form drafting. Different scene identifiers, pressures, support focuses, and event triggers must not collapse into one reused fallback packet, because that sameness propagates directly into repetitive chapter drafts later. 

@scene-courtroom-reversal refine
ref: $character-protagonist-001 -> Judge Mira Solari
should: keep the dialogue clipped and procedural at first
should: let the first memory gap arrive as a bodily disruption before it is interpreted intellectually
avoid: making the scene sound like legal exposition; the fear should leak through gesture and interruption.
