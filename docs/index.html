Project Overview

SCRIPTA Documentation

SCRIPTA is an implemented reference system for literary production. It transforms a book idea into a structured macro architecture, detailed chapter plans, symbolic micro plans, generated chapters, editorial outputs, and evidence-backed validation reports.

The public documentation surface is the SCRIPTA project itself: STG-guided planning, generation, validation, and metrics. Internal helper skills used to author the repository are not part of the SCRIPTA specification set.

Project objective

The project addresses a recurrent problem in creative writing: the rupture between macro narrative intention and micro narrative execution. SCRIPTA treats book production as a layered system in which the central idea, theme, structure, characters, worldbuilding, scenes, rhythm, and validation evidence remain connected from start to finish.

The planning files are not freeform notes. They are CNL documents built from command blocks such as @central-idea define, @chapter-004 define, or @suspense-ledger build, followed by fixed labeled lines and optional natural-language hints.

The first iteration of those files must be generated symbolically through CLI-style tooling with explicit parameters such as work size, genre profile, seed, and structural constraints. Later skill-mediated LLM stages refine those seed files, resolve placeholders, and add natural-language specificity without overwriting the original stage artifacts.

The current reference runtime is available through bin/scripta.mjs, covered by deterministic unit tests, and demonstrated through five generated short QA books under QA/.

BookWriter now emits dependency-free reader editions in English and Romanian as single HTML files with embedded cover art, table of contents, print styling, and machine-readable provenance. The QA root also exposes consolidated review artifacts such as qa-review.json and qa-tasks.md, plus directly browsable final books under QA/books/en/ and QA/books/ro/.

Core subsystems

The system is organized around planning, controlled generation, and verification. The planning stack turns theory into machine-readable contracts. The generation stack turns those contracts into chapters. The validation stack measures both literary quality and computational integrity.

Planning stack

MacroPlan, ChapPlan, and MicroPlan build the conceptual, chapter-level, and symbolic block-level representations of the book on top of a shared symbolic seed-generation layer.

Generation and review

CNLEnh, ChapGen, and BookWriter convert plans into readable chapters and editorial outputs without losing traceability to the planning stack.

Validation philosophy

SCRIPTA validation is not limited to a numerical dashboard. Every automated check must yield a normalized percentage score and a report with arguments, detected problems, and exact localization by chapter and paragraph or symbolic block.

The project bridges literary evaluation and computational evaluation. Narrative coherence, thematic depth, character complexity, originality, stylistic quality, emotional impact, interpretive openness, and cultural value coexist with metrics such as OI, NCS, CAD, EAP, TOP, VAD, CAR, CS, and NQS.

The DS set under docs/specs is the authoritative contract surface for the project.
