System Architecture

SCRIPTA Documentation

STG, short for Scripta Theoretical Guidelines, is the theoretical backbone of the project. It turns narrative theory into a staged production system that preserves coherence between concept, structure, chapter progression, prose generation, and evaluation.

STG is not treated as inspiration alone. It is the contract source for the SCRIPTA planning ontology, generation stages, validation procedures, and metrics.

Book workspace architecture

Each book must live in its own workspace. That workspace keeps planning artifacts, generated chapters, validation evidence, and exports together, which allows the system to compute metrics and explain them against exact locations in the manuscript.

Stage folder Primary owner Expected contents
phase1-macro/ MacroPlan Append-only macro seed artifacts: central idea, theme, wisdom, structure, narrative model, blueprint, characters, and worldbuilding.
phase2-chapters/ ChapPlan Append-only chapter seed artifacts with purpose, transitions, carry-forward pressure, and role in the whole-book arc.
phase3-micro/ MicroPlan Append-only symbolic block sequences, scene shapes, tension markers, and order constraints.
phase4-cnl/ CNLEnh Enhanced controlled-language successor artifacts with placeholder resolution and intended local effects.
phase5-drafts/ ChapGen Generated chapter drafts and chapter-level revision traces.
phase6-validation/ Validation suite Per-book summaries, issue inventories, stage audits, revision tasks, and raw validation data.
phase7-reports/ Metrics framework and validators Human-readable validation reports, stage review reports, and revision task reports.
phase8-exports/ BookWriter Single-file source reader editions, embedded cover art, manuscript sources, and source bundle manifests.
phase9-translations/ Translation Skill Translated HTML reader editions, chunk traces, and final publication bundle manifests.

Two-layer narrative model

The project is built on the STG distinction between macro and micro narrative construction. Macro planning handles central idea, theme, wisdom, structure, blueprint, characters, and worldbuilding. Micro planning handles scenes, sequences, chapters, action, conflict, dialogue, description, interior monologue, suspense, cliffhangers, pauses, and acceleration.

Macro layer

Defines the skeleton of the book, its causal logic, thematic tension, character arcs, and world rules.

Micro layer

Controls the concrete ordering of blocks, rhythm, immediacy, scene design, and local reader impact.

Execution strategy

SCRIPTA is intentionally hybrid. Scripts handle symbolic seed generation, deterministic planning, deterministic checks, metric normalization, and report assembly whenever the task can be made reproducible. LLMs are used only indirectly through skills that refine those prior-stage artifacts, draft chapters from them, and keep the stage lineage visible.

This division is not cosmetic. It is what allows the project to keep both generative flexibility and accountable validation while preserving append-only stage history.

The current runtime implements this architecture through the root CLI at bin/scripta.mjs, deterministic stage modules under runtime/, and generated QA workspaces for the five canonical baseline profiles.

CNL as the shared planning interface

The planning stack shares one textual contract: plan files are CNL documents composed of command blocks whose headers start with @identifier verb. This lets scripts scan plans quickly while still leaving room for richer NL hints that later LLM stages can exploit.
