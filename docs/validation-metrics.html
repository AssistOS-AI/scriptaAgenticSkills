Validation and Metrics

SCRIPTA Documentation

SCRIPTA combines literary criticism and computational verification. The system must score outputs on a normalized 0 to 100 scale and also explain each score through localized reports.

The reference runtime currently emits these reports and summaries for the five QA example books generated under QA/, one for each canonical baseline profile.

Validation now audits not only plans and drafts, but also the final HTML reader editions. Each run emits stage-level pass/warn/fail results together with a revision task list that can drive the next implementation pass.

The canonical QA run publishes the latest validation state as browsable HTML under QA/books/metrics/, so each demonstrator book has a human-readable dashboard in addition to its JSON summary.

Validation outputs

Every automated check must emit three surfaces: a normalized percentage, an evidence summary, and a list of issues localized to chapters and paragraphs or symbolic blocks.

Literary and computational layers

The framework keeps two complementary perspectives together. Literary indicators describe the text as an aesthetic and interpretive object. Computational indicators describe the text as the output of a controlled generation pipeline.

Literary indicators

Narrative coherence, thematic depth, character complexity, originality, stylistic quality, emotional impact, interpretive openness, and cultural value.

Computational indicators

OI, NCS, BCI, VAD, TOP, STG-CSG, SFSG, CCI, CAD, EAP, plus the composed metrics CAR, CS, and NQS.

Composed metrics

The project uses aggregated indicators to summarize the health of a manuscript without hiding the underlying evidence.

Metric Role Meaning
CARCompliance Adherence RateSummarizes ethical, legal, and bias-related robustness.
CSCoherence ScoreSummarizes structural following, continuity, character stability, and emotional arc quality.
NQSNarrative Quality ScoreSummarizes structural coherence together with literary quality dimensions.
